# PLAN-exhaustive-elevation.md

## Goal

Elevate the Agapay platform to a production-ready microfinance SaaS by hardening the multi-tenant architecture, implementing core socio-economic business logic, and achieving 100% Tagalog localization.

## 🔴 Architectural Decisions

1. **Stateless Database-Level Isolation (RLS)**: Implement PostgreSQL **Row-Level Security (RLS)** using `SET LOCAL app.tenant_id`. Every query from the application will be preceded by this session variable injection.
2. **Strict DB-Level Logic**: Move all loan math (Declining Balance, Penalty Caps) to **PL/pgSQL Stored Procedures**. The application layer will only call these functions, not perform the math itself.
3. **Comprehensive Audit Logs (Mutations & Reads)**: Implement a system to capture every `SELECT`, `INSERT`, `UPDATE`, and `DELETE` on sensitive tables (loans, repayments, wallets).
4. **Agapay Wallets**: Multi-purpose `SavingsAccount` models acting as internal wallets for disbursement and repayment.
