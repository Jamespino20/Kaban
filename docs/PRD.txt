[/] Cross-Tenant Leakage Check: Verify that a user cannot access loans, messages, or ledgers from a different tenant.
[/] Email Singularity Validation: Ensure the same email can register in multiple tenants but holds distinct trust scores and passwords per cooperative.
[/] Branch Switcher Integrity: Does switching branches correctly route the user and invalidate cross-tenant sessions securely?
[/] UI Branding Isolation: Confirm dynamic CSS tokens (brand_color, logo_url) inject safely without flashing or bleeding. [Can be drastically improved]

2. Member Onboarding & Verification
Secured multi-layer KYC and trust initiation.
[ ] KYC Document Upload: Validate handling of ID, Barangay Cert, Business Permit. [Does not validate on superadmin/admin side]
[/] Starter Tier Restriction: Ensure newly verified users strictly default to the "Starter" loan limit (₱5,000). [Unsure]
[/] Guarantor Endorsement Logic: Can a user legally invite/select guarantors only from their active tenant? (Also allows for same member, different tenant suggestions)
[ ] Admin Approval Flow: Verify the Tanaw queue accurately displays pending profiles without truncation. [Some approvals for superadmin, admin, and lender)

3. The Loan Lifecycle & Computation
Core financial pipeline from application to clearance.
[ ] Tier Checks: Reject loan applications exceeding the member's current Trust Tier limit.
[ ] Fixed vs Declining Balance Math: Ensure entry-level loans strictly use Fixed, while Trusted/Elite users are dynamically offered Declining Balance math.
[ ] Repayment Frequency Matrix: Validate computations correctly branch based on weekly, bi_weekly, and monthly frequencies.
[/] SOA Truth Extraction: Ensure the Statement of Account derives balances from business_ledger rather than isolated double-subtractions to prevent desync.
[ ] Edge Case: Partial repayments lower the principal but correctly handle remnant balances in subsequent schedules.

4. Default, Penalties & Guarantor Catchment
Ethical penalty caps and community accountability.
[ ] Penalty Rates Logic: Verify the exact mathematical steps (1-3 days: 2%, 4-7 days: 5%, 8-14 days: 8%, 15-30 days: 12%).
[ ] Maximum Penalty Cap: Test that penalties NEVER exceed 20% of the missed installment. (No compounding).
[ ] Guarantor Liability Trigger: Upon system declaration of "Default", ensure 20-30% liability is legally bound to the Guarantor's ledger.
[ ] Trust Graph Impact: Ensure both borrower and guarantor suffer localized (tenant-scoped) Trust Score drops.

5. Compassion Policy Enforcement (Has not been tested yet)
Ethical overrides for emergencies.
[ ] Approval Mechanism: Triggering a Compassion Action requires Admin verification.
[ ] Frequency Rule Enforcement: Guarantee the system hard-blocks a second Compassion use within the same loan cycle.
[ ] Action Integrity: Test "Grace Period" (shifting schedules without late hits), "Restructuring" (altering terms), and "Penalty Freeze".

6. Community, Mentorship & Messaging
The social engine of Agapay.
